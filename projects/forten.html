Forten

2020

Establishing a CSS framework for a team's web projects is a significant obstacle alone, because frameworks have their own particularities to acquaint oneself with. After experimenting with three separate frameworks, a proprietary framework came into perfect consideration, combining the best, most useful qualities of each one. Forten is a work-in-progress, with a completion goal in February 2021. This very site uses Forten!

At the beginning of designing my framwork, I used my experiences from trying out other frameworks to create four main pillars that would make my framework distinctive. The four pillars are: small filesize, no javascript(though I love Javascript), SCSS-based options and mixins, and B.E.M.-based class names.

A huge inspiration for Forten is a well-known CSS framework by the name of Milligram. It touts shockingly small file sizes for a full-fledged framework, but trial-runs demonstrated how it wouldn't fully support my future endeavors. Milligram's unminified CSS file is ~11KB, Bootstrap's uniminified CSS file sits at around ~198KB, while Forten's is currently ~21KB.

Drafting ideas for my framework began with a general idea of how it'd visually look. At first, I wanted to do a skeuomorphic style as the style is quite popular and eye-catching amongst the web design field so it'd complement my forward-looking preferences. I changed my mind on the visual elements because I was concerned with compatibility issues –particularly with forms– but going with a flatter, 2.5D aesthetic seems far more usable for more projects.
